User Guide

Overview

Jules Orchestrator schedules and runs AI agents against your repositories on a cron schedule. Each Task = one agent + one workflow pattern applied to one repo, fired automatically.

Upcoming Bar (top)

Shows the next 5 scheduled runs with a live countdown. Each slot displays: pattern · countdown · repo · exact time.

Creating a Task — field by field
Repository
GitHub repo in owner/repo format, e.g. QuoteSystemX/model-ML. Tasks with the same repo are grouped together on the board.
Agent
The AI persona that runs the task. Each agent has a different focus:
analyst code analysis & insights   orchestrator multi-step coordination   reviewer code review   debugger bug hunting   backend-specialist server-side   frontend-specialist UI/UX   security-auditor security   project-planner planning   test-engineer testing   crypto-go-specialist Go crypto
Mission
Free-text prompt given to the agent. Be specific: "Scan for SQL injection vulnerabilities and suggest fixes". The agent uses this as its primary instruction.
Workflow Pattern
Defines how the agent works through the repo:
discovery quick scan, surface findings   full_cycle plan → implement → review   reviewer diff review & comments   bugcatcher hunt for bugs   leakhound detect secret/data leaks   auditor compliance audit   featureforge implement a feature   refactor code clean-up   story_writer write user stories   sprint_planner plan sprint   sprint_closer close sprint & retro   test_engineer generate tests
Cron Schedule
Standard 5-field cron: min hour day month weekday.
0 */6 * * * — every 6 hours  |  0 9 * * 1 — every Monday 09:00  |  0 0 * * * — daily at midnight

The task ID is auto-composed as repo:agent:pattern and cannot be changed after creation. To change agent or pattern — delete and recreate.

Task Card — what each element means
Title / ID
Top line shows the pattern name (last segment of ID). Grey line below is the full ID repo:agent:pattern.
Status badge
PENDING waiting for next cron tick   RUNNING agent is executing right now   PAUSED manually suspended   SUCCESS last run succeeded   FAILED last run errored
Mission text
The prompt you set. Hover to see the full text if it's truncated.
Meta row
Clock icon + cron expression on the left. Bold pattern name on the right.
Logs
Opens execution history: each run shows status, duration, the full request sent to the LLM (IN), and the response (OUT).
Edit
Edit mission, schedule, or pattern. Agent and repo are fixed after creation.
/
Pause suspends future runs without deleting the task. Resume re-activates it.
Delete
Permanently removes the task and its execution history. Cannot be undone.
System Logs (bottom bar)

Click the bar to expand real-time server logs. Green lines = OK, red = errors, yellow = warnings. Auto-refreshes every 5 s.
